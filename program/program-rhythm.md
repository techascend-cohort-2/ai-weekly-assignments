# TechAscend Cohort 2 – Program Rhythm

TechAscend operates on structure and consistency.

This document explains how the program runs week to week.

The goal is not intensity.
The goal is disciplined, repeatable progress.

---

## Weekly Rhythm Overview

Every week follows the same structure:

- Monday → Assignment focus continues
- Wednesday → Learning Circles (Mandatory)
- Saturday → Workshop OR Blogging (Alternating)
- Sunday 11:59 PM → Assignment Deadline

Consistency matters.

---

## Wednesday – Learning Circles (Mandatory)

Learning Circles are small peer groups (3–5 students).

### Purpose
- Accountability
- Debugging support
- Concept clarification
- Weekly progress tracking

### Structure (60–90 minutes)

1. Each member shares progress
2. Group problem-solving or code discussion
3. Clarify confusing topics
4. Set commitments for the next week

### Expected Outcome
- Clear understanding of your progress
- Defined next steps
- Active engagement with peers

Attendance is required.

---

## Saturday – Alternating Structure

Saturdays rotate between:

### Week A → Workshop

Workshops are structured sessions led by:
- Program Lead
- Mentors
- Guest Speakers (when applicable)

Topics may include:
- Git deep dive
- Debugging strategy
- Project structure
- AI workflow
- Portfolio building
- Career readiness
- Communication skills

Purpose:
Accelerate your growth beyond weekly assignments.

---

### Week B → Mandatory Blogging

Every alternate Saturday is Blogging Day.

Each student must publish a short technical reflection.

### Where You Can Publish
- LinkedIn
- Medium
- Dev.to
- GitHub (as a structured README post)

### Minimum Content
- What you built this week
- What was challenging
- What you learned
- One example (code snippet or explanation)

### Purpose
- Improve technical communication
- Build public professional presence
- Document growth
- Strengthen confidence

Engineers who communicate clearly stand out.

---

## Weekly Assignment Cycle

Assignments follow a consistent rhythm:

- Work begins Monday
- Discuss progress Wednesday
- Reinforce learning Saturday
- Submit by Sunday 11:59 PM

Late submissions may be marked incomplete.

---

## Expectations

This program requires:

- Active participation
- Weekly output
- Consistent GitHub activity
- Respectful communication
- Professional discipline

Disappearing without communication is not acceptable.

If you are struggling:
- Ask early
- Use Slack
- Use your learning circle
- Contact a mentor

Communication prevents problems.
